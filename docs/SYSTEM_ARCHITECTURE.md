## 用語集（このシステムで使われている主な用語）

| 用語 | 読み方 | 意味 |
|------|--------|------|
| **React** | リアクト | 画面（ウェブページ）を作るための部品キット。Meta社が開発 |
| **SPA** | エスピーエー | 「シングルページアプリケーション」の略。ページ遷移なしで画面が切り替わるウェブアプリ |
| **Vite** | ヴィート | アプリのソースコードをブラウザで動く形に変換（ビルド）するツール |
| **TypeScript** | タイプスクリプト | JavaScript に型チェック機能を追加した言語。入力ミスを事前に防ぐ |
| **Tailwind CSS** | テイルウィンド シーエスエス | 画面のデザイン（色・余白・大きさ等）を指定するスタイル設定ツール |
| **shadcn/ui** | シャドシーエヌ ユーアイ | ボタン・表・入力欄などの見た目が整ったUIパーツ集 |
| **TanStack React Query** | タンスタック リアクトクエリ | サーバーから取得したデータを一時保存（キャッシュ）して高速表示する仕組み |
| **MSAL.js** | エムサル ジェイエス | マイクロソフトのログイン機能をウェブアプリに組み込むライブラリ |
| **Microsoft Entra ID** | エントラ アイディー | 旧称 Azure AD。マイクロソフトの認証基盤（社員のID・パスワード管理） |
| **OAuth 2.0** | オーオース 2.0 | 「このアプリにデータを見せてよいですか？」とユーザーに確認する認証の仕組み |
| **Graph API** | グラフ エーピーアイ | Microsoft 365のデータ（SharePoint, Teams等）を読み書きする窓口 |
| **SharePoint Online** | シェアポイント オンライン | Microsoft 365 のファイル・データ共有サービス。本システムのデータベース代わり |
| **GitHub Pages** | ギットハブ ページズ | GitHubが提供する無料のウェブサイト公開サービス。本システムの置き場所 |
| **GitHub Actions** | ギットハブ アクションズ | コードを変更したら自動でビルド＆公開してくれる仕組み（自動デプロイ） |
| **CI/CD** | シーアイ シーディー | 「継続的インテグレーション／継続的デリバリー」。コード変更→自動ビルド→自動公開の流れ |
| **JSON** | ジェイソン | データのやり取りに使う軽量な形式（例: `{"名前":"山田","年齢":30}`） |
| **HTTPS** | エイチティーティーピーエス | 通信を暗号化して盗聴を防ぐ安全な接続方式（URLが🔒マーク付き） |
| **トークン** | — | ログイン済みの証明書のようなもの。これを持っていればデータにアクセスできる |
| **環境変数** | — | パスワードやIDなどの秘密情報をコードの外に保管する仕組み |
| **デプロイ** | — | 作ったアプリを公開サーバーに配置して使えるようにすること |
| **ビルド** | — | 開発用のソースコードをブラウザで動く形に変換すること |
| **プロキシ** | — | 中継役。開発時にブラウザの通信を中継して認証を付加する |
| **IIS** | アイアイエス | Windows サーバーに標準搭載のウェブサーバーソフト |
| **Nginx** | エンジンエックス | 軽量・高速なウェブサーバーソフト（Linux で広く利用） |
| **SPA リライト** | — | どの URL にアクセスしても index.html を表示する設定。SPA に必須 |
| **アーカイブ** | — | 古いデータをファイルに待避（バックアップ）して、元のリストから削除すること |

### わかりやすく言うと

1. **ユーザーがブラウザでアプリを開く** → GitHub Pages または社内サーバーからアプリの画面が届く
2. **マイクロソフトアカウントでログイン** → Entra ID がログインを確認し通行証を発行
3. **データの読み書き** → 通行証を見せて Graph API 経由で SharePoint のデータを操作
4. **Teams 通知** → 「発報」ボタンで Graph API 経由で Teams チャネルにメッセージ投稿（ChannelMessage.Send 権限）

### GitHub Pages と社内サーバーの違い

| 項目 | GitHub Pages | 社内サーバー |
|------|-------------|------------|
| 環境変数 | GitHub Secrets → ビルド時注入 | `.env.production.local` に直書き |
| ベースパス | `/manmarudairyreport/` | `/`（設定可能） |
| ビルド | GitHub Actions 自動 | `npm run build:server` 手動 |
| HTTPS | 自動 | サーバー側で設定 |
| 更新方法 | `git push` で自動 | `git pull` → 手動ビルド → 配置 |

## 9. セキュリティ対策

| 対策 | やっていること |
|------|-------------|
| **ログイン認証** | マイクロソフトアカウント必須。未ログインではデータを見られない |
| **権限管理** | ログインしたユーザー本人の権限でのみデータ操作（管理者権限は不要） |
| **秘密情報の保護** | パスワードやID等はコードに直書きせず、環境変数と GitHub Secrets で管理 |
| **通信の暗号化** | すべての通信が暗号化（HTTPS）されており、盗聴されない |
| **コードの分離** | 開発用リポジトリは非公開。公開用は秘密情報を含まないクリーンなコードのみ |

## 10. データアーカイブ（古いデータの待避と削除）

### SharePoint リストの制限

| 制限 | 値 | 影響 |
|------|----|------|
| リスト最大アイテム数 | 3,000万件 | ほぼ問題なし |
| **ビュー閾値** | **5,000件** | **超えるとクエリが遅くなる** |

> 日報が1日10人分の場合、年間約3,650件。**2年弱で5,000件に到達**するため、
> 年1回のアーカイブが推奨。

### アーカイブの仕組み

```
  古いデータ（1年以上前）          archives/ フォルダ
       │                              │
       │  ① JSON ファイルに            │
       │     エクスポート（待避）        │
       │  ────────────────────────>   │
       │                              │  Reports_20270413_120000.json
       │  ② SharePoint リストから       │  Plans_20270413_120000.json
       │     削除                      │
       │                              │
```

### 使い方（scripts/archive_old_data.py）

| コマンド | 動作 |
|----------|------|
| `python3 scripts/archive_old_data.py --dry-run` | 何が消えるか確認だけ（安全） |
| `python3 scripts/archive_old_data.py --export-only` | JSON に待避のみ（削除しない） |
| `python3 scripts/archive_old_data.py` | 1年以上前のデータを待避＆削除 |
| `python3 scripts/archive_old_data.py --months 6` | 6ヶ月以上前で実行 |

### 対象リスト

| リスト | アーカイブ対象 | 理由 |
|--------|:---:|------|
| 作業報告（Reports） | ✅ | 日々増えるデータ |
| 作業予定（Plans） | ✅ | 日々増えるデータ |
| 顧客マスタ | ❌ | 件数が少なく変動しない |
| システムマスタ | ❌ | 件数が少なく変動しない |
| 作業種別マスタ | ❌ | 件数が少なく変動しない |

### 推奨運用スケジュール

| タイミング | 操作 | コマンド |
|-----------|------|---------|
| 毎月1回 | 件数確認 | `--dry-run` で現状把握 |
| 年1回（期末など） | 1年以上前のデータを待避＆削除 | `python3 scripts/archive_old_data.py` |

待避した JSON ファイルは `archives/` フォルダに年月日付きで保存されます。
必要に応じて社内の共有フォルダやクラウドストレージにバックアップしてください。
